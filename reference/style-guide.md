# 快速浏览 PPT 风格指南

Agent 生成销售讲义时必须参考此文档，确保**结构**和**视觉风格**与目标一致。

## 一、目标参考文件

- **若用户提供了目标 PPT 样例**（如 `【快速浏览】xxx.pdf`）：以其为视觉与结构范本。
- **否则**：以 `reference/【快速浏览】有赞支付解决方案+2025年最新版.pdf` 为默认范本。

生成前务必**打开目标 PDF 浏览**，理解其布局与风格。

## 二、封面结构

```
┌─────────────────────────────────────────────────────────┐
│                    主标题（大号粗体）                      │
│                    副标题（中号）                         │
│                                                         │
│     slogan1        slogan2        slogan3               │
│  （全渠道收款）  （智能高效分账）  （门店/生意增长）        │
│                                                         │
├─────────────────────────────────────────────────────────┤
│ 杭州有赞科技有限公司 © 2012-2026 All Rights Reserved.   │
│ 香港上市代码: 08083.HK                                   │
└─────────────────────────────────────────────────────────┘
```

- 主标题：产品/方案名
- 副标题：核心价值一句话
- 三行 slogan：并列关键词，用空格或竖线分隔
- 页脚：统一版权信息，灰色小字

## 三、典型幻灯片布局模式

### 模式 A：对开页（问题 vs 方案）

左右分栏，左「问题/现状」，右「解决方案/价值」：

```
┌──────────────────────┬──────────────────────┐
│ 你的现有模式是否这样？ │ 有赞支付助力你实现     │
│                      │                      │
│ [流程图/示意图]       │  [标签+说明]          │
│ 消费者→门店→商家     │  极稳通道：xxx        │
│                      │  专业护航：xxx        │
│ 你是否面临以下问题？  │  灵活适配：xxx        │
│ • 风险频发：xxx       │  持牌经营：xxx        │
│ • 模式单一：xxx       │  顶级防护：xxx        │
└──────────────────────┴──────────────────────┘
```

- 左侧：现状描述 + 痛点列举（短句）
- 右侧：价值点用**标签+一句话**（如「极稳通道」「专业护航」）
- 标签 4–6 个，每标签下 1 行说明

### 模式 B：产品页

```
┌─────────────────────────────────────────────────────────┐
│ 产品名    有赞支付核心产品①    标签1  标签2  标签3       │
├─────────────────────────────────────────────────────────┤
│ 产品介绍&价值点                                           │
│                                                         │
│ 产品定义                                                 │
│ 一段话说明产品定位...                                    │
│                                                         │
│ 核心价值                                                 │
│ • 统一收：xxx                                            │
│ • 稳到账：xxx                                            │
│ • 账对清：xxx                                            │
│                                                         │
│              [右侧：流程图/架构示意图]                    │
└─────────────────────────────────────────────────────────┘
```

- 标题行：产品名 + 序号 + 3 个短标签
- 产品定义：1 段话
- 核心价值：3–4 条 bullet，格式「标签：说明」
- 右侧可配简化流程图（文字描述即可，无需真实图）

### 模式 C：一图看懂（架构/流程图）

```
┌─────────────────────────────────────────────────────────┐
│ 一图看懂有赞支付                                          │
│ 一句话总结定位...                                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  [中心：商家账户]                                        │
│     ↑                                                    │
│  公域付款  私域付款  到店付款                             │
│     ↑         ↑         ↑                               │
│  分账场景 | 分账规则 | 商家账户                           │
│  (多列关键词)  (多列)   (总部/门店/供应商...)              │
│                                                         │
│ 底部：合规持牌 | 资金全保 | 数据盾牌 | 金融级系统         │
└─────────────────────────────────────────────────────────┘
```

- 中心概念突出，外围分块（场景、规则、账户）
- 用文字描述流程，关键词分行/分列
- 底部 4 个合规标签

### 模式 D：业务需求示意图

```
┌─────────────────────────────────────────────────────────┐
│ 商家典型业务需求①  纯收款                                 │
├─────────────────────────────────────────────────────────┤
│ 消费者 → 门店 → 有赞收款 → 商家银行账户                  │
│ （用文字描述资金流，无需真实图形）                        │
│ 商家只有收款需求，没有分账需求                            │
└─────────────────────────────────────────────────────────┘
```

- 标题：需求类型 + 简短命名
- 资金流用「→」连接关键节点
- 底部一句总结

## 四、视觉规范

| 元素     | 规范 |
|----------|------|
| 标题     | 粗体，每页一个主标题 |
| 要点     | 3–6 条/页，每条 10–25 字 |
| 标签     | 2–4 字短词 + 冒号 + 说明 |
| 页脚     | 每页统一：杭州有赞科技有限公司 © 2012-2026... |
| 留白     | 不堆砌，每页聚焦一个信息块 |
| 对比     | 问题与方案、现状与价值，用左右或上下对比 |

## 五、底料 → 幻灯片的映射

| 底料结构           | 目标幻灯片类型     |
|--------------------|--------------------|
| 1. 主标题 + 定义   | 封面 + 模式 A 对开页 |
| 1.1 1.2 子章节    | 模式 B 产品页 或 模式 C 架构图 |
| 表格（利益相关方） | 模式 A 左右对比 或 精简为标签 |
| 表格（产品定义）   | 模式 B 产品页 |
| 💥 核心逻辑        | 模式 A 右侧价值点 |
| 🌟 分账场景        | 模式 C 分账场景块 或 模式 B 核心价值 |
| 业务模式图         | 模式 D 业务需求示意图 |

## 六、页面类型与视觉元素（详见 visual-analysis.md）

| 类型 | 视觉元素 |
|------|----------|
| 封面 | 渐变、3D 插画、平台 logo、slogan 标签 |
| 对开页 | 流程图（圆/箭头）、问号/对勾图标 |
| 产品页 | 标题栏、产品定义、核心价值、右侧流程图 |
| 业务需求流程图 | 圆形节点、箭头、色块分组 |
| 关系图 | 中心节点、四周实体、箭头+标签 |
| 三列问题-方案 | 绿/蓝/橙色块、侧边标签 |
| 结尾 | 全屏背景图、白字叠加 |

## 七、生成前检查清单

- [ ] 已打开目标 PDF 浏览整体风格
- [ ] 已阅读 [visual-analysis.md](visual-analysis.md) 理解逐页结构
- [ ] 封面含主标题、副标题、slogan、页脚
- [ ] 问题/方案类内容用对开页布局
- [ ] 产品类用产品页布局（定义 + 核心价值 + 标签）
- [ ] 架构/流程用「一图看懂」布局
- [ ] 每页页脚统一
- [ ] 要点精简，无大段文字
